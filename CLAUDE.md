# CLAUDE.md — Юр-бот «URIST2026»

Telegram-бот с ИИ по юридическим вопросам для «синих воротничков» (те, кто ищет
работу: охранники, водители и т.п.). Отвечает на бытовые рабочие вопросы (как
проверить работодателя, что делать если не платят, штрафы на работе). Тариф
«Базовый»: веб-поиск + ответы + память диалога. Оплата пакетами запросов через
ЮKassa. Бот: [@URIST2026_1_BOT](https://t.me/URIST2026_1_BOT).

## Старт сессии (обязательно, okplan)

Перед любой работой:
1. Прочитать `docs/STATUS.md` — текущий этап, активная работа, блокеры.
2. `git tag -l "stage-*"` — сверить с колонкой «Закрывающий тег» в STATUS. Тег без записи в таблице = STATUS устарел, сначала починить.
3. `git log --oneline -10` — что менялось последним.
4. Открыть `docs/07_ROADMAP.md` на текущем этапе — там чекбоксы открытых acceptance-пунктов.

Прогресс — по методологии **okplan** (три слоя правды: STATUS.md → чекбоксы ROADMAP → git-теги `stage-N-done`). Тег ставится только после живой верификации человеком. Не отмечать чекбоксы и не ставить теги наперёд. **За сессию — один этап, потом отдаём человеку на проверку.**

## Стек (зеркалим club_bot)

Python 3.12 · aiogram 3 · Postgres 16 (**отдельная схема `urist_bot`** — на ПК куча чужих проектов, свои таблицы не трогаем) · Redis (FSM + кеш + память диалога) · Loguru · Alembic (автоприменение миграций при старте) · APScheduler (polling платежей ЮKassa) · Docker/Compose · hot-reload в dev · polling (без вебхуков).

**ИИ (OpenRouter):**

| Слой | Модель |
|---|---|
| FSM / служебные решения / уточнения | `google/gemini-2.5-flash-lite` |
| Финальный ответ + веб-поиск (tool-calling) | `google/gemini-2.5-flash` |

**Веб-поиск (инструменты агента):** Tavily (основной) · Exa (запасной) · Firecrawl (запасной). API-ключи поисковиков меняются из `/admin` и хранятся в БД (`app_settings`), применяются горячо.

## Экономика (валюта — «запросы», не токены)

- **1 запрос = 1 доставленный ответ ИИ по существу** (с источниками). Списывается в **одной точке** — момент выдачи финального ответа.
- **Бесплатно для юзера**: уточняющие вопросы (лимит ≤2–3), весь веб-поиск и внутренняя цепочка, а также отмена / таймаут / «не по теме» / ошибка ИИ.
- **Защита баланса владельца (OpenRouter)**: капы — ≤2 уточнения, ≤3–4 шага поиска, лимит токенов; служебка на дешёвой модели.
- Юзер покупает пакеты **10/20/30** запросов (+ N бесплатных на старте). Цена за 1 запрос конфигурируется в `/admin` (дефолт 1).

## Правила работы (из глобального CLAUDE.md)

- **Полный слой логирования (Loguru)** — приоритет заказчика: каждое действие пользователя (входящие, кнопки, команды) — дата/время, username, user_id, first_name, текст ответа бота, на русском. Чем полнее логи, тем быстрее чиним баги.
- **HTML-форматирование** текстов бота (`<b>`, `<i>`, `<u>`, `<code>`).
- **Навигация под аудиторию 35–50 лет**: продуманная, без незаконченной логики — «где есть вход, там есть и выход» (везде кнопки Назад / Главное меню, нет тупиков).
- **При поиске ИИ пишет, что ищет** информацию, а не молчит.
- Без эмодзи в копирайте/UI-текстах (текстовые маркеры → · — допустимы). Эмодзи-галочки из навигации заказчика сохраняем как в источнике.
- `.bat`-файлы запуска: первой строкой `chcp 65001 > nul`.
